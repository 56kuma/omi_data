# APIトークン管理とユーザー切り替えの確認

会話では、あるシステム（ディレクター画面のような管理UI）でのAPIトークン管理の方法について説明している。画面の一番下にある「トークンの発行」ボタンを押すとリフレッシュされ、新しいトークンが再発行されてしまうため、このボタンは基本的に押さないように注意喚起している。また、現在は自分自身のユーザーとしてAPIを叩いているが、本来は複数ユーザーでAPIを利用したいという要件があり、その場合は別ユーザーのトークンを使ってアクセスする必要があることを確認している。さらに、6番目のトークンなど、画面上には見えていないが保存済みのトークンが存在していること、特定の値をディレクター側に設定していることで自動接続や自動実行が行われるため、誤って操作すると意図せぬ動作につながる点も共有されている。
